Gitlet

The little brother of Git written in Java.

Functionality

With the Gitlet version control system, you can:

  • Initialize a Gitlet repository in a local directory.
  • Save the directory through commits.
  • Create and maintain multiple branches.
  • Restore a file or entire commit.
  • View the global commit history as well as the one in your branch.
  • Merge changes made between branches.
  • View your current Gitlet status.

Demo

Source code can be provided on request

  • First, initialize a Gitlet repository using the init command. This creates a hidden .gitlet folder in the specifed directory, which will keep track of all of your commits.
  • Now, you can start checking your Gitlet status using the status command.
  • Try creating a new branch using  branch [branch name].

  • Stage files for addition using add [file name].
  • Commit the current directory using commit [message] so that its files can be retrieved later.
  • View the commit history log of the current branch using log.

  • Stage a file for removal using rm [file name].
  • Checkout a different branch using checkout [branch name] in the working directory activate [branch name]
  • Merge branches using merge [branch name] and report modification conflicts.

  • Completely reset to a past commit through reset [commit id]
  • Remove a branch pointer using rm-branch [branch name]
